# Product Spec：StockMind

版本：1.0　日期：2026-06-22　状态：Implemented

## 1. 问题与目标

普通 LLM 股票问答把事实、推断和建议混在一段文本中，缺少专家交叉检查、历史连续性和运行证据。StockMind 的目标是产生一份可追踪、可降级、明确风险边界的教学研究报告，而不是自动交易或收益承诺。

## 2. 用户故事

- 作为学生，我希望无 API Key 也能完成稳定的五分钟演示。
- 作为研究者，我希望量化、舆情与风控观点独立生成并由 CIO 汇总。
- 作为评阅者，我希望查看每个节点是否成功、耗时多少以及历史记忆是否生效。
- 作为运维者，我希望单次 LLM 故障产生降级结果，不泄漏 Key，也不让进程无解释崩溃。

## 3. 范围

包含：6 位 A 股代码、日期校验、演示数据工具、并行专家分析、CIO 报告、跨会话检索记忆、Tracing、CLI、Docker、测试。暂不包含：实盘交易、自动下单、收益预测承诺、用户账户体系和生产行情授权。

## 4. 功能验收标准

| ID | 需求 | 可执行验收 |
|---|---|---|
| FR-01 | 输入校验 | 非 6 位数字或倒序日期返回明确错误 |
| FR-02 | 多智能体 | 量化、舆情、风控均生成结果后 CIO 才运行 |
| FR-03 | 记忆 | 同代码第二次运行的 `memory_context` 非空 |
| FR-04 | 安全边界 | 最终结果包含“不构成投资建议” |
| FR-05 | 可观测 | 每次成功运行至少产生 6 个 trace 事件 |
| FR-06 | 离线演示 | 无 Key、无网络可完成端到端运行 |

## 5. 非功能指标

- Demo 成功率目标 100%（5 次本地基准）。
- Demo 平均耗时目标 < 1 秒（不含解释器启动）。
- API Key 只从环境变量读取，日志不记录提示词、密钥和完整敏感输入。
- 任一 LLM 专家失败时返回显式降级文本，并记录错误类型。

## 6. 非目标与伦理

系统不提供个性化投资顾问服务，不接券商交易接口，不用未来数据评估历史决策。模拟数据字段均带 `demo_fixture` 标签，避免被误认为实时行情。
